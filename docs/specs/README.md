# Milestone specs

Each milestone gets a **pre-flight spec** here at `M{N}.md`, written *before*
coding starts — not after, not "as we go". The spec is the canonical statement
of what the milestone is *for*, and is the first milestone-specific doc a
re-anchored session reads (after `CLAUDE.md`).

A spec is **frozen once coding begins**. Deviations are recorded in the
implementation plan (`docs/milestones/M{N}.md`) or as ADRs
(`docs/decisions/`), never by editing the spec retroactively.

Copy `TEMPLATE.md` to start a new spec.
