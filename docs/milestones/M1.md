# M1 — Implementation plan

Plan for delivering the M1 spec (`docs/specs/M1.md`). The plan records *how*
M1 is built and any deviations from the frozen spec.

## Approach

Build the app in dependency order: config → backend client → auth → shared UI →
caregiver screens → patient screens → diagnostics → docs/memory system.

## Steps

1. **Project config** — `package.json`, `app.json`, `tsconfig.json`,
   `babel.config.js`, `.gitignore`, `.env.example`, `expo-env.d.ts`.
2. **Backend + theme** — `lib/theme.ts`, `lib/supabase.ts` (publishable key,
   AsyncStorage, auto-refresh), `lib/branding.ts`, `lib/format.ts`.
3. **Auth** — `lib/auth.tsx` (AuthProvider/useAuth, session lifecycle).
4. **Data access** — `lib/queries.ts` (RLS-scoped reads; row types mirror the
   web Drizzle schema; `pingBackend` probe).
5. **Shared UI** — `components/Screen.tsx`, `components/DisclosureFooter.tsx`,
   `components/ui.tsx`.
6. **Routing + auth gate** — `app/_layout.tsx`, `app/index.tsx`,
   `app/(auth)/`.
7. **Caregiver mode** — `app/(caregiver)/_layout.tsx` (banner + gate),
   `patients.tsx`, `patient/[id].tsx`, `diagnostics.tsx`.
8. **Patient mode** — `app/(patient)/_layout.tsx`,
   `view/[id]/accounts.tsx`, `view/[id]/account/[accountId].tsx`.
9. **Handoff system** — `docs/`, `scripts/re-anchor.{sh,ps1}`, `memory/`,
   `CLAUDE.md`, `README.md`; `git init`.

- (none yet) — record any here with a one-line rationale as they happen.
